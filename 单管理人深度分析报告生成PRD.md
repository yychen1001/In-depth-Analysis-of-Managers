# 单管理人深度分析报告生成 PRD

**文档版本**：v1.0（三件套交付版）  
**适用场景**：基于公开数据资料、管理人路演材料、策略介绍、投资观点、业绩数据等素材，生成可用于机构配置决策的单管理人深度分析报告。  
**目标产物**：一套逻辑连贯、事实严谨、配置视角明确、可复用到不同管理人的标准交付件，包括 Markdown 源报告、HTML 交互网页和 DOCX 正式文档三件套。

---

## 一、项目背景与核心目标

单管理人研究报告的价值不在于复述管理人材料，而在于回答配置方真正关心的问题：这家管理人的能力来源是什么，策略本质是什么，收益来自哪里，在哪些环境中有效，何时可能失效，适合放在组合中的什么位置，以及后续应如何跟踪。

本次庄贤投资报告的迭代证明，一个高质量管理人分析流程应先完成“理解管理人”而非“整理材料”。管理人的路演、月报、策略介绍和业绩数据只是原材料，研究报告需要在充分扫描材料后形成独立分析框架，将团队基因、策略逻辑、业绩路径、风险控制和配置价值纳入同一条主线。

本 PRD 的目标是将该流程产品化，使其可以高质量复刻到其他管理人，同时保留对不同策略类型的适配能力。最终交付不以单一文本为终点，而以内容一致、格式统一、可阅读、可流转、可下载的 `MD + HTML + DOCX` 三件套为 1.0 标准。

## 二、用户与使用场景

### 2.1 目标用户

- 财富管理、FOF、家办、机构销售、投资顾问等需要评估私募管理人的专业用户。
- 需要将管理人纳入组合配置池、观察池或尽调备忘录的投资研究人员。
- 需要将非结构化管理人材料转化为标准化但有洞察力报告的资管机构。

### 2.2 核心使用场景

- 新管理人初筛：判断是否值得进入进一步尽调。
- 已关注管理人复盘：结合业绩数据和材料更新配置观点。
- 组合配置决策：明确管理人的组合角色、适配环境、配置边界和跟踪指标。
- 内部投委会沟通：形成语言克制、逻辑严谨、结论清晰的管理人分析材料。

## 三、产品定位

该产品不是“材料摘要工具”，而是“管理人研究分析生成工具”。它必须在素材基础上形成有判断力的机构报告。

产品应具备四项能力：

1. **素材吸收能力**：系统扫描路演、投资观点、策略介绍、业绩数据等不同类型文档，提取事实、观点、案例、规则和业绩路径。
2. **策略理解能力**：从材料中识别管理人的真实投资范式，而非停留在管理人自我标签。
3. **配置分析能力**：从投资人视角评估收益来源、环境适配、风险边界和组合用途。
4. **报告迭代能力**：支持围绕结构、标题、表格、语言风格、网页展示和 Word 输出进行多轮精修。
5. **三件套交付能力**：以 Markdown 作为内容母版，同步生成 HTML 网页和 DOCX 文档，确保三端内容一致、版式统一、下载可用、验收可复核。

## 四、输入范围与素材处理要求

### 4.1 标准输入

| 输入类型 | 内容示例 | 主要用途 |
|---|---|---|
| 路演交流 | 管理人自述、问答、团队背景、策略解释、案例复盘 | 识别团队基因、真实表达、能力圈和认知方式 |
| 策略介绍 | 投资框架、风控规则、产品定位、案例展示 | 提炼策略流程、组合管理和风险控制 |
| 投资观点/月报 | 市场判断、持仓方向、阶段复盘 | 识别市场认知、环境适配和策略执行一致性 |
| 业绩数据 | 年化收益、波动、回撤、年度收益、回撤修复 | 分析收益路径、状态依赖、风险收益特征 |
| 产品资料 | 产品线差异、合同约束、工具使用、仓位限制 | 判断产品间差异是否来自策略、工具或风险预算 |

### 4.2 素材扫描原则

扫描素材时必须区分五类信息：

- **硬事实**：时间、履历、产品数据、回撤区间、风控规则、案例事件。
- **管理人观点**：对市场、行业、策略和自身失误的解释。
- **研究线索**：反复出现的关键词、方法论、能力圈和偏好。
- **可验证案例**：成功案例、失败案例、阶段性承压案例。
- **待审慎处理内容**：营销性表述、未验证判断、过度归因和不可外推业绩。

扫描完成后，应形成内部研究笔记，而不是立即写报告。研究笔记至少包括：团队基因、策略本质、产品线、业绩路径、案例库、风控规则、适配环境、失效模式、配置假设和待核实问题。

## 五、核心分析方法

### 5.1 先定义投资范式

每份报告都应先提炼管理人的投资范式，用一句公式化表达压缩其策略本质：

`管理人策略 = 能力基础 + Alpha 来源 + 组合实现 + 风险约束`

示例：  
`庄贤大周期策略 = 化工产业认知 + 供需拐点识别 + 个股利润兑现 + 交易纪律约束。`

该表达不是口号，而是全文分析主线。后续团队、产品、业绩、案例、风险和配置价值都应围绕该范式展开。

### 5.2 从“事实”走向“解释”

报告不得停留在材料分点堆积。每个重要事实后都应回答其含义：

- 团队履历说明了什么能力来源？
- 产品线差异是策略差异，还是同一策略下的风险预算差异？
- 高收益来自持续 Alpha，还是特定环境红利？
- 连续亏损是个股错误、风控不足，还是策略传导链失效？
- 风控规则能控制什么风险，又不能控制什么风险？
- 管理人适合作为核心底仓、增强模块、卫星配置，还是观察型策略？

### 5.3 把业绩作为压力测试，而不只是展示结果

业绩分析必须超越年化收益和最大回撤。重点应包括：

- 收益是否集中在少数年份。
- 回撤发生在什么市场环境中。
- 回撤修复耗时对投资人体验意味着什么。
- 代表产品是否能反映核心策略。
- 不同产品线之间是否真的分散，还是同源风险的不同弹性表达。
- 高收益年份是否可重复，还是来自阶段性环境红利。

对于连续亏损、长回撤或修复较慢的阶段，应作为策略压力测试进行分析。例如庄贤 2022-2023 年连续亏损的分析重点不是“当年表现不好”，而是“供需改善、利润增长和股价表现之间的传导链条在资金净流出、估值下移和风格偏离环境中被削弱”。

### 5.4 案例用于解释能力圈与边界

实践案例不应作为附录式材料堆放，而应服务于投资框架解释：

- 成功案例用于验证能力来源、信号识别、交易执行和利润兑现。
- 失败案例用于识别能力边界、风控执行和策略失效条件。
- 阶段性承压案例用于解释市场环境与策略表现之间的关系。

案例分析应保持中立。不能只写成功案例，也不能把失败案例处理成管理人自我辩护。案例结论必须回到“能力圈是否有效、边界在哪里、后续如何跟踪”。

## 六、推荐报告结构

不同管理人应根据策略类型调整结构，但建议使用以下基础框架。

### 投资摘要

投资摘要不编号，位于正文前。必须先给出投资范式公式，再用数段文字回答核心判断：

- 管理人是谁，策略本质是什么。
- 配置价值来自哪里。
- 主要优势是什么。
- 关键风险和局限是什么。
- 适合什么资金，不适合什么资金。

摘要应直接、克制、结论清晰，不设置“核心判断”等多余二级标题。

### 一、团队背景与核心人员

分析团队不是罗列履历，而是解释履历如何塑造策略：

- 创始背景、组织基因、产业资源或平台来源。
- 核心人员能力互补关系。
- 团队稳定性、关键人依赖、研究资源宽度。
- 能力圈边界及外延扩张风险。

### 二、产品线与业绩分析

本章应融合产品定位和业绩数据，避免割裂。

产品线部分先说明不同产品的核心差异：策略是否相同、仓位是否不同、工具是否不同、风险预算是否不同、目标客户是否不同。

业绩部分至少展示两个表：

- 产品核心指标表：起始日、年化收益、年化波动、最大回撤、回撤期、修复耗时、定位。
- 年度收益表现表：用于观察收益分布、周期性、状态依赖和回撤路径。

表格应横向压缩空间，居中、紧凑、不换行。

业绩解读应覆盖：

- 代表产品的样本意义。
- 产品间收益弹性差异。
- 长回撤与修复周期。
- 收益分布是否均匀。
- 业绩的进一步解读，即业绩背后的策略含义。

### 三、投资框架与实践案例

本章回答“管理人如何赚钱”。

可展开为：

- 框架总览：从信号到标的，再到组合兑现。
- 研究流程：数据、调研、事件、模型、验证、复盘。
- 选股或建仓框架：买什么、为什么买、何时买。
- 卖出与止盈止损：何时承认错误，何时兑现收益。
- 实践案例：成功、失败和承压案例同时呈现。

### 四、策略定位与环境适配

本章回答“这个策略在什么环境有效”。

应包括：

- 策略不是管理人自称标签，而是研究者重新定义后的本质定位。
- 策略适配环境。
- 策略不利环境。
- 策略 Alpha 的条件性。
- 与传统同类策略的差异。

### 五、风险控制

风险控制不能只写“重视风控”。必须具体拆解：

- 仓位控制。
- 单票集中度。
- 单票止损。
- 组合回撤管理。
- 止盈纪律。
- 回撤复盘机制。
- 流动性和规模容量。

同时必须分析风控边界：风控能降低个股尾部风险和执行错误，但不一定能对冲策略 Beta、行业系统性风险、估值压缩和市场风格偏离。

### 六、配置价值与策略边界

本章从配置方视角给出结论：

- 组合角色：核心底仓、卫星增强、周期弹性、低波替代、绝对收益替代等。
- 配置前提：投资人需要接受的回撤、波动和持有周期。
- 策略边界：能力圈、环境、规模、产品相关性、收益外推。
- 跟踪指标：后续应观察哪些变量来判断策略是否继续成立。

### 七、投资总结

总结应清晰直接，不重复正文。应回答：

- 管理人是否具备配置价值。
- 配置价值来自哪里。
- 主要局限是什么。
- 推荐以何种方式使用。
- 哪类资金应谨慎。

## 七、标题与结构规范

### 7.1 标题原则

标题应具备解释力，不应只是材料标签。

较弱标题：  
`业绩分析`、`投资理念`、`案例介绍`

较好标题：  
`业绩特征解读`、`策略定位：周期框架中的供需错配与成长弹性`、`实践案例：能力圈有效性与边界`

二级标题数量应适中。章节过多会显得零散，章节过少会降低阅读效率。最终报告建议控制在七个一级章节左右，二级标题用于增强阅读节奏。

### 7.2 结构连贯性

报告不能简单填充素材。每章之间应形成递进关系：

团队背景解释能力来源；产品和业绩验证策略结果；投资框架解释收益机制；案例验证能力圈；环境适配解释什么时候有效；风险控制说明如何约束失败；配置价值给出组合用途；投资总结形成结论。

## 八、语言与风格规范

### 8.1 专业风格

语言应符合专业资管机构报告风格：

- 客观、中性、克制。
- 避免营销化、故事化和过度修辞。
- 用判断性语言，但不夸大确定性。
- 优势和局限同时呈现。
- 所有结论尽量落到配置含义。

### 8.2 禁用或慎用表达

避免出现以下不专业表达：

- “材料显示”
- “补充数据表明”
- “根据材料可见”
- “该管理人非常优秀”
- “未来大概率持续高收益”
- “显然”“必然”等过度确定性表达

更好的表达方式是直接陈述事实与分析：

- “锐进2号自2018年以来年化收益24.39%，最大回撤-33.13%。”
- “该结果说明策略具备较强收益弹性，但净值体验高度依赖周期环境。”
- “配置时应将其视为周期弹性资产，而非组合稳定器。”

### 8.3 事实边界

所有内容必须基于材料和公开数据资料。可以形成推论，但必须让推论与事实之间保持合理距离。不得编造管理规模、持仓、人员、案例或未来判断。

如果材料不足，应在报告中弱化结论，或列为后续尽调问题。

## 九、不同策略类型的适配方式

基础框架可复用，但不同管理人应调整重点。

| 管理人类型 | 分析重点 | 需要特别关注 |
|---|---|---|
| 主观股票多头 | 团队能力圈、选股框架、组合集中度、回撤控制、风格暴露 | 收益是否来自个股 Alpha 还是风格 Beta |
| 周期/资源类股票 | 供需框架、价格信号、库存、政策、事件驱动 | 环境适配、失效模式和回撤修复 |
| 量化指增 | 模型框架、因子暴露、交易成本、超额稳定性、规模容量 | 超额衰减、风格暴露和换手成本 |
| 市场中性 | 对冲方式、基差、杠杆、风控、收益来源 | 极端行情下对冲失效和流动性 |
| CTA/期货策略 | 趋势/套利/基本面逻辑、品种分散、回撤结构 | 震荡市失效、杠杆与止损机制 |
| 多策略 | 子策略贡献、风险预算、相关性、资金分配 | 真实分散度和管理复杂度 |
| 债券/固收+ | 久期、信用、杠杆、票息、权益增强 | 信用尾部风险和流动性 |

## 十、交付形态要求

标准交付形态为三件套：`Markdown 源报告 + HTML 交互网页 + DOCX 正式文档`。三件套不是三个独立版本，而是同一份研究内容的不同载体。Markdown 承担内容母版功能，HTML 承担阅读和展示功能，DOCX 承担正式流转和归档功能。

三件套命名应保持一致，仅文件后缀不同。例如：

- `某管理人深度分析报告.md`
- `某管理人深度分析报告.html`
- `某管理人深度分析报告.docx`

若需要托管为网页站点，应额外提供 `index.html` 作为站点根入口。`index.html` 不承载报告正文，只负责将仓库或站点根路径跳转到正式报告 HTML，避免外部用户必须访问中文长文件名。

三端同步原则：

- 内容以 Markdown 为准，HTML 和 DOCX 不得自行增加、删减或改写正文结论。
- HTML 可以增加目录、进度条、返回顶部、下载按钮等阅读交互，但不应增加正文之外的分析模块。
- DOCX 可以根据正式文档需要优化页边距、字体、标题、表格和两端对齐，但不得改变报告逻辑。
- 每次正文内容修改后，必须同步检查 HTML 内嵌正文和 DOCX 内容是否一致。

### 10.1 Markdown 报告

Markdown 是主报告源文件，应保证结构清晰、表格规范、便于后续转网页或 Word。

格式要求：

- 全局字体可设置为微软雅黑。
- 表格居中、紧凑、不换行。
- 投资摘要前保留研究对象、研究范围、数据截止。
- 研究范围统一使用“公开数据资料”。
- 关键结论可以适当加粗，但必须基于真实结论判断，不能机械加粗关键词。
- 可使用少量图文摘要、策略评价表、产品选择表、风险拆解表等结构化图表提升可读性。
- 图表必须服务于理解，不得替代正文分析，也不得引入材料之外的新事实。

### 10.2 网页版本

网页应严格按照深度报告呈现，不应在正文前额外增加产品卡片、环境速览等多余模块，除非用户明确要求。

网页 1.0 标准功能：

- 顶部品牌标题，建议使用克制、机构化表达，例如 `In-depth Analysis of Managers`。
- 自动识别一级和二级标题生成目录。
- 目录当前阅读位置轻微高亮。
- 桌面端使用左侧目录栏；目录栏可折叠和展开，折叠按钮应位于目录与正文之间，不遮挡目录和正文。
- 桌面端目录栏可拖动调整宽度，正文区域随目录宽度自动适配。
- 平板和手机端目录改为顶部粘性目录区，支持折叠和横向滑动，避免左侧目录占用正文空间。
- 目录文字不换行，超长标题使用省略号。
- 顶部阅读进度条。
- 右下角返回顶部按钮。
- 下载 Word 按钮。
- 下载 MD 按钮。

网页视觉风格应克制、专业，避免喧宾夺主。交互应服务阅读，而不是抢夺报告主体。

网页 1.0 视觉与交互约束：

- 网页正文必须严格跟随深度报告，不在开头额外增加产品线速览、环境速览、营销式导语等模块。
- 主体字体使用微软雅黑，正文两端对齐，段落间距适度紧凑。
- 表格居中、紧凑、不换行，适合横向展示业绩数据。
- 关键内容仅使用加粗，不使用底色、高亮块或过度装饰。
- 折叠/展开按钮背景透明、颜色淡、形状克制，保持始终可见，不随正文或目录滚动消失。
- 目录拖拽条视觉上可以很淡，但交互命中区应足够宽，避免难以拖动。
- 网页按钮区应简洁，1.0 标准仅保留 `下载WORD`、`下载MD`，不保留字号调整按钮和打印按钮。
- 手机端顶部按钮应使用两列网格或等效紧凑布局，确保按钮不溢出、不重叠、不遮挡标题。
- 平板端应适当收窄目录宽度和页面间距，保证正文阅读区宽度。
- 移动端表格应支持横向滚动，不强行压缩造成文字重叠。
- 下载按钮应直接指向同名 `.docx` 和 `.md` 文件，并在点击后给出简短状态提示。
- 若网页需要外部托管，根目录必须保留 `index.html` 入口，并确保根链接能够直接进入报告页。

### 10.3 Word 版本

Word 输出应支持正式流转：

- DOCX 格式。
- 微软雅黑。
- 正文两端对齐。
- 表格居中、紧凑、不换行。
- 标题层级清晰。
- 保留免责声明。

DOCX 应与 Markdown 和 HTML 内容一致。若报告正文发生调整，应重新生成 DOCX，并至少完成文件可打开、表格未明显错位、标题层级正常、免责声明保留的基础验收。

## 十一、迭代流程

本类报告需要多轮迭代，不能假设一次成稿。

### 11.1 第一轮：深度理解与初稿

目标是形成完整分析，而非追求排版完美。

重点检查：

- 是否理解管理人策略本质。
- 是否形成独立框架。
- 是否覆盖团队、产品、业绩、框架、风险和配置价值。
- 是否有成功和失败案例。
- 是否有环境适配和策略边界。

### 11.2 第二轮：结构重组

根据阅读体验调整章节顺序和层级。

典型调整包括：

- 投资摘要置前且不编号。
- 团队背景与核心人员合并。
- 产品线与业绩分析合并。
- 策略本质与配置价值、环境适配融合。
- 章节数量压缩，避免零散。

### 11.3 第三轮：洞察增强

补充真正提升报告价值的分析：

- 连续亏损或大回撤的结构性解释。
- 收益状态依赖。
- Alpha 条件性。
- 策略失效模式。
- 产品线相关性。
- 高收益年份是否可外推。

### 11.4 第四轮：语言与呈现

统一专业风格，删除不专业表达，优化标题长度和层级。

重点包括：

- 删除“材料显示”“数据表明”等表述。
- 提高标题一致性。
- 保持结论中立。
- 压缩过度叙事。
- 优化表格排版。

### 11.5 第五轮：多格式输出

在内容质量基本定稿后，生成标准三件套：Markdown、HTML、DOCX。除非另有要求，PDF 不作为 1.0 标准产物，可以由 Word 或浏览器另行转换获得。

本轮目标不是重新写报告，而是完成同一内容在不同载体上的稳定呈现。所有格式应与主报告内容保持一致，不能出现正文漂移。

重点包括：

- 以 Markdown 作为内容母版，确认标题、表格、加粗、免责声明均已定稿。
- 将 Markdown 同步嵌入或渲染到 HTML，确保网页正文与报告一致。
- 生成 DOCX，确保正式文档可打开、可流转、可归档。
- HTML 与 DOCX 均保留同名文件下载逻辑或文件对应关系。
- 如需网页托管，生成或保留 `index.html` 根入口。
- 表格在 HTML 和 DOCX 中保持紧凑、不换行、居中展示。
- 若发现网页或 Word 版式问题，优先修正渲染层，不随意改写研究内容。

### 11.6 第六轮：交互与格式验收

三件套生成后必须进行交付验收，避免“内容正确但使用失败”。

网页验收重点：

- 顶部按钮仅保留 `下载WORD`、`下载MD`。
- 下载 Word 和下载 MD 指向同名文件，点击后能触发下载或打开下载流程。
- 桌面端左侧目录可点击跳转，正文滚动时目录对应模块高亮。
- 桌面端目录栏可折叠、展开，按钮始终可见，且不遮挡目录和正文。
- 桌面端目录栏可拖动调整大小，正文区域随宽度变化自动适配。
- 平板和手机端目录以顶部粘性目录区呈现，可折叠、可横向滑动。
- 手机端下载按钮不换行、不溢出、不遮挡品牌标题。
- 移动端表格可横向滚动，正文和表格不发生重叠。
- 右下角返回顶部按钮存在并可用。
- 页面不出现与报告正文无关的额外速览模块。
- 页面无明显乱码、文本重叠、表格换行失控或按钮遮挡。
- 托管版根链接可通过 `index.html` 直接进入报告页。

Word 验收重点：

- DOCX 文件可正常打开。
- 正文字体、段落、标题层级和表格样式符合正式报告要求。
- 正文两端对齐，表格居中紧凑。
- 与 Markdown 正文内容一致。

Markdown 验收重点：

- 文件作为最终内容母版可独立阅读。
- 关键结论加粗逻辑准确，不误导、不泛化。
- 图表和正文互相支撑，不造成事实新增或含义偏移。

## 十二、质量验收标准

一份合格的单管理人深度分析报告必须满足以下标准：

| 维度 | 验收标准 |
|---|---|
| 事实准确性 | 关键数据、人员、产品、日期、案例均有材料依据 |
| 策略理解 | 能用一句投资范式概括管理人策略本质 |
| 逻辑连贯 | 团队、策略、业绩、案例、风险和配置结论相互支撑 |
| 洞察深度 | 至少包含收益来源、失效模式、环境适配和策略边界分析 |
| 配置视角 | 明确说明适合什么资金、承担什么风险、放在组合什么位置 |
| 客观中立 | 优势和局限同时呈现，不做营销化表达 |
| 语言质量 | 专业、克制、无明显口语化或材料堆砌痕迹 |
| 表格展示 | 核心业绩数据充分展示，紧凑清晰 |
| 可复用性 | 框架可迁移，但内容根据管理人特征定制 |
| 输出一致性 | Markdown、HTML、DOCX 内容一致 |
| HTML 可用性 | 目录、折叠、桌面拖拽、返回顶部、下载 Word、下载 MD 均可用 |
| 响应式兼容 | 桌面、平板、手机均可正常阅读，按钮、目录、表格不溢出、不遮挡 |
| DOCX 可流转性 | 文件可打开，版式正式，正文两端对齐，表格清晰 |
| Markdown 母版性 | Markdown 可作为后续更新和批量复刻的唯一内容母版 |

### 12.1 三件套最终验收清单

每个管理人报告交付前，应完成以下检查：

- 文件是否齐全：`.md`、`.html`、`.docx` 三个文件均存在，且文件名一致。
- 内容是否一致：三端标题、章节、核心结论、表格、免责声明一致。
- 研究范围是否统一：使用“公开数据资料”。
- 投资摘要是否位于正文前，且不占章节序号。
- 投资范式是否在摘要开头明确呈现，且贯穿全文。
- 业绩数据是否以至少两个核心表格充分展示。
- 关键结论是否选择性加粗，且没有底色或误导性高亮。
- HTML 是否无额外正文外分析模块。
- HTML 目录是否不换行，桌面端折叠/展开与拖拽是否不遮挡正文。
- HTML 平板和手机端目录是否可折叠、可横向滑动，按钮是否适配小屏。
- HTML 是否提供下载 Word、下载 MD 功能。
- 托管场景是否保留 `index.html` 根入口。
- DOCX 是否可打开、可流转、版式整洁。

## 十三、常见失败模式

需要避免以下问题：

- 把管理人材料拆成条目后重新排列，缺乏研究判断。
- 只写优势，不写局限和策略边界。
- 只展示业绩表，不解释业绩路径。
- 对高收益做线性外推。
- 把产品线差异误写成策略差异。
- 只写成功案例，不写失败或承压案例。
- 风控部分停留在原则，不分析风控能否真实约束风险。
- 章节过多，标题零散，阅读路径不清楚。
- 网页交互过度，削弱报告正式感。
- HTML 开头额外加入与正文无关的产品速览、环境速览或营销导语。
- HTML 折叠按钮、拖拽条、返回顶部按钮遮挡目录或正文。
- HTML 在手机或平板上按钮挤压、目录占屏、表格文字重叠。
- 移动端仍沿用桌面左侧目录，导致正文阅读空间不足。
- HTML 下载按钮存在但无法指向实际文件。
- 删除 `index.html` 导致站点根链接无法直接进入报告。
- Word、HTML 和 Markdown 内容不一致。
- 只更新 Markdown，忘记同步 HTML 内嵌正文或重新生成 DOCX。

## 十四、后续可扩展能力

未来可进一步产品化为半自动研究生成流程：

- 素材清单自动识别。
- 业绩数据自动转表。
- 投资范式候选生成。
- 策略类型自动匹配报告框架。
- 成功/失败案例自动抽取。
- 环境适配矩阵生成。
- 配置价值和跟踪指标模板化。
- Markdown、HTML、DOCX 一键同步输出。
- `index.html` 根入口自动生成。
- HTML 目录、拖拽、折叠、移动端粘性目录、下载按钮组件模板化。
- DOCX 样式模板化，自动继承标题、表格和正文格式。

但无论自动化程度如何，最终报告仍需保留研究判断环节。管理人深度分析的核心不是生成文字，而是形成对管理人能力、边界和配置价值的可检验判断。
